# Information Architecture (IA)

## StartSmart Cafe Franchise

AI 기반 프랜차이즈 카페 창업 현실 검증 시뮬레이터 (MVP)

---

## IA 설계 원칙

1. **분석은 백스테이지, 판단 결과는 프런트스테이지**
2. 사용자는 "계산"이 아니라 **"결론과 이유"를 본다**
3. AI는 버튼이 아니라 **판단 레이어**로 자연스럽게 노출된다
4. 한 화면 = 한 질문에 답한다

---

## 전체 구조 요약 (1 Depth)

1. 브랜드 선택
2. 입지 선택 (지도)
3. 조건 입력
4. 분석 실행
5. 결과 요약 대시보드
6. AI 판단 상세
7. 시뮬레이션 비교
8. 리포트 출력

---

## 1. 브랜드 선택

### 목적

* 분석 대상 프랜차이즈 확정

### 구성 요소

* 브랜드 카드 리스트 (12개)

  * 브랜드명
  * 포지션 태그 (저가 / 중가 / 베이커리 등)
* 선택 CTA: "이 브랜드로 분석하기"

---

## 2. 입지 선택 (지도)

### 목적

* 상권 및 물리적 입지 확정

### 구성 요소

#### 2.1 지도 인터랙션

* 지도 확대/축소
* 핀 클릭 → 좌표 확정
* 반경 선택: 300m / 500m / 1km

#### 2.2 상권 요약 카드

* 반경 내 경쟁 카페 수
* 경쟁 밀도 지표 (Low / Mid / High)

#### 2.3 **AI 로드뷰 분석관 (Physical Risk Analyzer)**

* 로드뷰 이미지 자동 로딩
* AI 분석 결과 카드

  * 간판 가림: 낮음 / 중간 / 심각
  * 급경사 여부
  * 층위 추정 (1층 / 1.5층 / 반지하)
  * 보행 가시성 평가

> *데이터로 확인 불가능한 현장 물리 리스크를 AI가 시각적으로 분석*

---

## 3. 조건 입력

### 목적

* 개인 창업 조건 반영

### 구성 요소

#### 3.1 기본 입력

* 초기 투자금
* 월세
* 평수
* 점주 근무 여부 (Toggle)

#### 3.2 판매량 설정

* AI 제안 판매량 (보수 / 기대 / 낙관)
* 사용자 목표 판매량 입력

> *AI 제안에는 반드시 "왜 이 수치인지" 근거 문구 포함*

---

## 4. 분석 실행

### 목적

* 계산 및 AI 판단 트리거

### 구성 요소

* "이 조건으로 분석하기" CTA
* 로딩 단계 표시

  1. 손익 계산
  2. 상권 경쟁 분석
  3. 로드뷰 물리 리스크 분석
  4. AI 리스크 해석

---

## 5. 결과 요약 대시보드 (Front Stage)

### 목적

* 한눈에 판단 가능하게 요약

### 핵심 카드

* 최종 점수 + 신호등
* 예상 생존 개월 수
* 회수 개월
* 월 순이익

### 차트

* 손익 구조 스택 차트
* 민감도 ±10% 그래프

---

## 6. AI 판단 상세 (Consulting Layer)

### 목적

* 왜 이런 결과가 나왔는지 설명

### 구성 요소

#### 6.1 핵심 리스크 Top 3

* 구조적 리스크 설명
* 발생 원인 요약

#### 6.2 개선 제안

* 효과 큰 순서대로 3가지 제안
* 각 제안의 기대 변화 요약

#### 6.3 경쟁 환경 해석

* 경쟁 강도 수준
* 차별화 가능성
* 가격 전략 방향

---

## 7. 시뮬레이션 비교

### 목적

* 조건 변경에 따른 결과 비교

### 구성 요소

* Before / After 카드
* 점수 / 회수 개월 변화 강조

---

## 8. 리포트 출력

### 목적

* 공유 가능한 결과물 제공

### 구성 요소

* PDF 다운로드

  * 요약 페이지
  * 차트 페이지
  * AI 판단 코멘트 페이지

> *PDF는 "컨설턴트 보고서" 톤 유지*

---

## 9. Phase 2 (확장)

### AI 심층 상담관 (Post-Analysis Advisor)

* 결과 기반 질의응답 전용 인터페이스
* 해당 분석 컨텍스트로만 응답
* 자유 채팅이 아닌 판단 보조 중심

---

## IA 한 줄 요약

> **StartSmart의 IA는 사용자가 계산을 조작하는 구조가 아니라,**
> **AI가 먼저 판단을 제시하고 사용자는 선택하는 구조로 설계된다.**
