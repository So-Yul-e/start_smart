# 서버 로그 확인 방법 (업데이트)

## 문제 상황

프론트엔드에서 `status: undefined`가 계속 나오고, 서버 콘솔에 로그가 찍히지 않는 문제가 발생했습니다.

## 해결 방법

### 1. 서버 로그 파일 확인

서버를 실행한 터미널에서 다음 명령어로 로그를 확인할 수 있습니다:

```bash
tail -f server.log
```

또는 프로젝트 루트 디렉토리에서:

```bash
cat server.log
```

### 2. 서버 상태 확인

서버가 제대로 실행되고 있는지 확인:

```bash
curl http://172.16.48.47:3000/api/brands
```

정상 응답이 오면 서버는 실행 중입니다.

### 3. 분석 결과 조회 테스트

분석 ID를 알고 있다면 직접 테스트:

```bash
curl http://172.16.48.47:3000/api/result/analysis_17698661918...
```

응답에 `status` 필드가 포함되어 있는지 확인하세요.

## 추가된 디버깅 로그

서버 응답에 더 자세한 로그가 추가되었습니다:

- `[결과 조회] 분석 상태: ...` - DB에서 가져온 상태와 result 존재 여부
- `[결과 조회] 분석 완료, 결과 반환 (DB status: ..., result 타입: ...)` - 완료 시 상세 정보
- `[결과 조회] 응답 데이터: ...` - 실제로 보내는 응답 데이터 (처음 200자)

## 문제 해결 체크리스트

1. ✅ 서버가 실행 중인지 확인
2. ✅ `server.log` 파일 확인
3. ✅ `/api/result/:analysisId` 직접 테스트
4. ✅ 응답에 `status` 필드가 포함되어 있는지 확인
5. ⏳ 프론트엔드에서 받은 응답 확인 (브라우저 콘솔)

## 다음 단계

1. 서버를 재시작하고 분석을 실행
2. `server.log` 파일에서 `[결과 조회]` 로그 확인
3. 브라우저 네트워크 탭에서 실제 응답 확인
4. 문제가 지속되면 로그를 공유하여 추가 디버깅
