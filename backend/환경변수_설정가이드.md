# 환경변수 설정 가이드

## 백엔드 서버 환경변수

### `.env` 파일에 추가할 내용

```bash
# 서버 포트
PORT=3000

# 서버 호스트
# 개발 환경: 특정 IP로 바인딩 (보안상 더 안전)
# 백엔드 담당자가 자신의 로컬 IP를 확인하여 설정
HOST=<로컬IP>

# 또는 모든 네트워크 인터페이스에서 접근 (0.0.0.0)
# ⚠️ 보안상 취약할 수 있으므로 개발 환경에서만 사용
# HOST=0.0.0.0
```

**보안 고려사항:**
- `HOST=0.0.0.0`: 모든 네트워크 인터페이스에서 접근 가능 → **보안상 취약**
- `HOST=<로컬IP>`: 특정 IP로만 바인딩 → **더 안전** (권장)
- `HOST=localhost` 또는 `HOST=127.0.0.1`: 로컬에서만 접근 → **가장 안전** (프론트엔드 개발자와 다른 기기에서 접근 불가)

**보안 권장사항:**
- ✅ **`HOST=<로컬IP>`**: 특정 IP로만 바인딩 → **보안상 안전** (권장)
- ⚠️ `HOST=0.0.0.0`: 모든 네트워크 인터페이스에서 접근 가능 → **보안상 취약** (개발 환경에서만 사용)
- 🔒 `HOST=localhost`: 로컬에서만 접근 → **가장 안전** (다른 기기 접근 불가)

### 프론트엔드 개발자에게 알려줄 정보

프론트엔드 개발자가 사용할 API URL은 **환경변수가 아닌** 백엔드 담당자가 직접 알려줘야 합니다:

```bash
# 현재 로컬 IP 확인
ipconfig getifaddr en0  # Mac
# 출력: <로컬IP>

# 프론트엔드 개발자에게 전달 (직접 알려주기, 문서에 하드코딩하지 않음)
API URL: http://<로컬IP>:3000
```

프론트엔드 개발자는 이 IP를 자신의 환경변수나 설정 파일에 넣어야 합니다.

## 프론트엔드 환경변수 (프론트엔드 개발자가 설정)

### 방법 1: HTML에서 설정 (정적 파일인 경우)

```html
<script>
  // 백엔드 담당자가 알려준 IP 주소 (환경변수로 관리)
  window.API_BASE_URL = 'http://<로컬IP>:3000';
</script>
<script src="config.js"></script>
```

### 방법 2: .env.local 파일 (React 등 빌드 도구 사용 시)

```bash
# frontend/.env.local (Git에 커밋하지 않음)
REACT_APP_API_URL=http://<로컬IP>:3000
```

## 현재 IP 확인 방법

```bash
# Mac
ipconfig getifaddr en0

# Windows
ipconfig | findstr IPv4

# Linux
hostname -I
```

## 주의사항

1. **IP 주소는 변경될 수 있습니다**: WiFi 재연결 시 IP가 바뀔 수 있으므로, 서버 실행 전에 확인하세요.

2. **.env 파일은 Git에 커밋하지 마세요**: `.gitignore`에 이미 포함되어 있습니다.

3. **프론트엔드 개발자에게 IP 전달**: 백엔드 담당자가 직접 알려줘야 합니다. 문서에 하드코딩하지 마세요.
