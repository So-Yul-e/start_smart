# PostgreSQL 비밀번호 설정

## 현재 상황
- `.env` 파일: `DB_PASSWORD=postgres1234`
- PostgreSQL 실제 비밀번호: 알 수 없음 (기본값이거나 다른 값)

## 해결 방법

### 방법 1: PostgreSQL 비밀번호를 `.env`에 맞게 변경 (권장)

```bash
# PostgreSQL 접속 (비밀번호 없이 접속 가능한 경우)
psql -U postgres

# 또는 sudo로 접속
sudo -u postgres psql
```

접속 후:
```sql
ALTER USER postgres WITH PASSWORD 'postgres1234';
\q
```

### 방법 2: `.env` 비밀번호를 PostgreSQL에 맞게 변경

1. PostgreSQL의 실제 비밀번호 확인
2. `.env` 파일에서 `DB_PASSWORD` 값을 실제 비밀번호로 수정

### 방법 3: 비밀번호 없이 접속 가능한 경우

일부 PostgreSQL 설치에서는 localhost에서 비밀번호 없이 접속 가능합니다.
이 경우 `.env`의 비밀번호를 빈 문자열로 설정하거나, `pg_hba.conf` 설정을 확인하세요.

## DBeaver 연결 시

- **Password**: `.env`의 `DB_PASSWORD` 값과 동일하게 입력
- 또는 PostgreSQL에 설정된 실제 비밀번호 입력

## 비밀번호 확인 방법

```bash
# PostgreSQL 접속 시도
psql -U postgres -d startsmart

# 비밀번호를 물어보면 현재 비밀번호 입력
# 접속 성공하면 비밀번호가 맞는 것
```

## 비밀번호 변경 후

1. `.env` 파일의 `DB_PASSWORD`와 PostgreSQL 비밀번호가 일치하는지 확인
2. DBeaver 연결 정보도 동일하게 설정
3. 연결 테스트
